研究方向Research

主要研究面向下游任务的数据清洗，目前的工作大致分为三个方向。I mainly work on task-aware data cleaning; the work currently falls into three directions.

01

需求驱动的数据清洗Demand-driven data cleaning

研究清洗动作怎么跟着下游任务走：哪些错误值得修，哪些可以留着不动。代表工作是 DemandClean（VLDB 2025）。

How cleaning actions should follow the downstream task: which errors are worth fixing and which can be left alone. Representative work: DemandClean (VLDB 2025).

02

可追踪的算子执行Operator-grounded execution

智能体的清洗决策落到具体的 cleaner、规则和执行日志上，结果可以复跑，出问题可以回滚。对应开源的 DemandPrep artifact。

Agent decisions are grounded in concrete cleaners, rules, and execution logs; results can be rerun and bad changes rolled back. Released as the DemandPrep artifact.

03

脏数据上的 ML / AutoMLML and AutoML over dirty data

预算有限时，花在清洗上的开销会影响模型训练和 AutoML 搜索，反过来也一样。DMCO（ICML 2026）研究两边的预算怎么分配。

Under a limited budget, what goes into cleaning affects model training and AutoML search, and vice versa. DMCO (ICML 2026) studies how to split the budget between the two.

DemandPrep 的执行流程The DemandPrep execution loop

Sense 识别数据错误和任务需求 Detect errors and task demand
Allocate 选择 no-op / repair / delete / replace Choose no-op, repair, delete, or replace
Execute 调用可复现的清洗算子 Run reproducible cleaning operators
Verify 按下游指标决定提交或回滚 Commit or roll back based on downstream metrics

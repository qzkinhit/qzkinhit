研究主线Research Lines

把数据清洗从“修数据”推进到“按任务决策”。From fixing data to making task-conditioned data decisions.

01

需求驱动的数据清洗Demand-driven cleaning

用下游模型收益、数据真实性、数据多样性和清洗成本共同约束清洗动作，避免把“越干净越好”当作唯一目标。

Cleaning actions are selected under downstream utility, authenticity, diversity, and cost instead of optimizing only intrinsic data neatness.

02

可追踪的算子执行Operator-grounded execution

把智能体的策略选择落到可复现的 cleaner、规则、执行日志和回滚机制上，让系统行为能被检查、比较和复跑。

Agentic decisions are grounded in reproducible cleaners, rules, execution traces, and rollback paths so that results can be inspected and rerun.

03

脏数据上的 ML / AutoMLML and AutoML over dirty data

研究有限预算下清洗、建模和 AutoML 搜索之间的相互影响，让学习系统在真实不完美数据上更稳健。

I study the interaction among cleaning, modeling, and AutoML search under limited budgets for more reliable learning over imperfect data.

Artifact

A demand-driven data preparation artifact with action allocation, operator-grounded execution, real traces, and an interactive demo.

 RL Cleaning

DemandClean

DQN agent 在 per-cell 粒度选择 no-op、repair 或 delete，平衡真实性、多样性和模型收益。

A DQN-based cleaner that selects no-op, repair, or delete per cell while balancing authenticity, diversity, and model utility.

包含真实脏数据、清洗输出、日志、baseline 对比和评估脚本的 benchmark 风格仓库。

A benchmark-style repository with native dirty data, cleaned outputs, logs, baseline comparisons, and evaluation scripts.
